# Data-entry forms use Angular Signal Forms

Data-entry forms (settings pages, login) are built with **Angular Signal Forms** (`@angular/forms/signals`, stable `@publicApi 22.0`): a typed model `signal`, `form(model, schema)`, and `[formField]` bindings on the vendored spartan controls, which implement the `FormValueControl`/`FormCheckboxControl` contracts (`hlmInput`, `hlm-checkbox`, `hlm-select`). This replaces the previous template-driven approach (`FormsModule` + `[(ngModel)]` with native HTML validation). We chose it over both the incumbent template-driven forms and Angular reactive forms because the model is a plain typed signal — no `FormGroup` ceremony, no `ControlValueAccessor` glue — validation and derived state (`disabled`, `hidden`) are colocated reactive rules, and it composes with the signal-based state services the rest of the UI already uses. The cost is that Signal Forms is a newer API surface and the whole suite of forms moves onto it at once; we accept that because it is stable in Angular 22 and the spartan controls we vendor already support `formField`, so there is no bridge to maintain.

Validation is **client-side and synchronous only** — schema validators (`required`, `email`, `pattern`, `min`/`max`, `validateTree`, …), no `validateAsync`/`validateHttp`. Field errors render inline by looping `field().errors()` into `hlm-field-error` within an `hlmField`, and the submit button is disabled while `f().invalid()`. The `<form>` is bound with the `[formRoot]` directive and the submit action is registered through `form()`'s `submission` option; `FormRoot` intercepts the native submit, prevents default, and runs the action only when the form is valid (marking it touched otherwise, so errors surface). The action calls `api.x()` and reloads the list on success. The **server** round-trip outcome — the class of error validation cannot catch (auth rejection, a 500, a duplicate the backend enforces) — is reported through the `Feedback` toast seam, never mapped onto fields; this keeps the ADR-0007 boundary intact (inline = client-side pre-submit, toast = server outcome) and refines 0007's earlier note that the login auth error lived inline. Filter/search boxes are deliberately out of scope: a lone search string bound to a signal is not a form, so they keep `FormsModule` + `[(ngModel)]="signal"` rather than wrapping a single value in `form()`.
