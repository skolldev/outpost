# No egress restrictions on Notification Channels

Outpost places no restrictions on Notification Channel destination URLs: private IP ranges, localhost, and internal hostnames are all permitted, payloads are not HMAC-signed, and stored URLs are shown unmasked to Admins. This is a deliberate rejection of multi-tenant SaaS SSRF defenses, which would break a primary on-premises use case — posting to internal receivers such as self-hosted automation tools — while protecting against an attacker who, in a single-tenant Installation, would already hold Admin access. The webhook URL itself is treated as the bearer credential, the same trust model used by Teams and Slack incoming webhooks. This posture is sound only because Notification Channel management is restricted to Admins; if channel configuration is ever opened to Members or to per-Project self-service, egress filtering and payload signing must be revisited.
